# docs/

All non-code knowledge. Documentation leads code — nothing gets built in `src/` or `products/` without its doc existing first.

- `company-blueprint/` — business layer (canonical for the platform)
- `builders-handbook/` — how to build here (read first)
- `platform-architecture/` — technical architecture, one file per concern
- `agent-specifications/` — canonical agent spec + templates
- `adrs/` — decision records
- `implementation/` — platform-level plans (product PRDs live under `products/*/docs/`)
- `assets/` — diagrams, sources, glossary
